# Mnemos Project Template

A simple reusable project-memory template for long-running work with GPT, GitHub, and human collaborators.

This repository is intended to become a GitHub Template Repository. New projects can be created from it, then initialized for software, worldbuilding, genealogy, research, writing, nonprofit work, or private personal projects.

## Purpose

The repository is the durable memory. GPT is the reasoning engine.

This template exists so a future GPT session or human collaborator can quickly answer:

```text
Where did we leave off?
What matters?
What is canonical?
What should happen next?
```

## Start Here

For any project created from this template, read these first:

1. `IDENTITY.md`
2. `CURRENT_STATE.md`
3. `PROJECT_MAP.md`
4. `docs/INDEX.md`
5. `docs/process/BOOKKEEPING_PROTOCOL.md`

## Core Files

- `IDENTITY.md` — project bootloader and privacy/profile summary.
- `CURRENT_STATE.md` — active handover and next step.
- `PROJECT_MAP.md` — repository navigation and authority hierarchy.
- `MNEMOS_VERSION.md` — template/system version.
- `docs/INDEX.md` — documentation navigation.
- `docs/constitution/PROJECT_CONSTITUTION.md` — stable project laws and boundaries.
- `docs/memory/PROJECT_JOURNAL.md` — durable decisions and rationale.
- `docs/process/BOOKKEEPING_PROTOCOL.md` — startup and end-of-session maintenance.
- `prompts/` — reusable GPT prompts.

## Basic Workflow

Start a session with:

```text
Catch up on this project. Read IDENTITY.md, CURRENT_STATE.md, PROJECT_MAP.md, and the relevant indexes before proposing work.
```

End a significant session with:

```text
Ok, let's do our bookkeeping.
```

## Privacy Rule

Create separate private repositories for separate private projects. Invite collaborators only to the repositories they need.

Do not mix unrelated private work in one repository unless the project deliberately requires it.
