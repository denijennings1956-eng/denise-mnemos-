# Memory Index

## Purpose

This folder preserves the evolving memory of the project.

Use it for journal entries, decision history, lessons learned, rejected alternatives, and continuity notes that future sessions should be able to find.

## Core Files

- `PROJECT_JOURNAL.md` — the main project journal and seed entry template.

## What Belongs Here

Use this folder for:

- significant project decisions;
- why-decisions;
- rejected paths;
- experiments;
- lessons learned;
- session summaries;
- continuity notes;
- future search terms;
- evidence of how the project evolved.

## What Does Not Belong Here

Do not use this folder for settled laws or stable principles. Those belong in `docs/constitution/`.

Do not use this folder as a dumping ground for all project content. Working files should live in the relevant project folders and be linked from journal entries when needed.

## Findability Rule

Every important journal entry should include:

- title;
- timestamp;
- status;
- tags;
- keywords;
- aliases or old terms;
- related files;
- related entries;
- next steps.
