# Constitution Index

## Purpose

This folder preserves the stable principles and accepted rules of the project.

The constitution is not for every idea. It is for principles that have been accepted and should guide future work.

## Core Files

- `PROJECT_CONSTITUTION.md` — the project constitution and constitutional seed.

## What Belongs Here

Use this folder for:

- accepted principles;
- durable rules;
- project boundaries;
- stable standards;
- non-negotiable practices;
- core definitions.

## What Does Not Belong Here

Do not use this folder for experiments, loose ideas, brainstorming, or unresolved questions. Those belong in `docs/memory/PROJECT_JOURNAL.md` until they are accepted.

## Graduation Rule

A principle should usually appear in the journal before it graduates into the constitution.

When a principle is promoted, link back to the journal entry that explains why.
