# Docs Index

This folder contains the main documentation layers for the project.

## Start Here

- `../IDENTITY.md` — project bootloader.
- `../CURRENT_STATE.md` — active handover.
- `../PROJECT_MAP.md` — root navigation and authority.
- `constitution/PROJECT_CONSTITUTION.md` — stable project laws and accepted principles.
- `memory/PROJECT_JOURNAL.md` — durable reasoning, decisions, rejected paths, and lessons learned.
- `process/BOOKKEEPING_PROTOCOL.md` — startup and end-of-session maintenance.

## Areas

- `constitution/` — accepted project laws, boundaries, terminology, and decision standards.
- `memory/` — journal entries, rationale, lessons learned, and findable project memory.
- `process/` — workflows for GPT sessions, initialization, and bookkeeping.

## Guidance

Keep important context findable.

Prefer links, tags, timestamps, related files, and search terms over long unstructured prose.

If an idea is promising but untested, put it in the journal. Promote it to the constitution only after it proves useful.
